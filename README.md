# SCT_ML_02_Credit_Card_Fraud_Detection

## Overview
This project is an end-to-end Machine Learning pipeline to detect fraudulent credit card transactions. It handles data preprocessing, exploratory data analysis (EDA), and predictive modeling using multiple machine learning algorithms.

## Dataset
The dataset is the [Credit Card Fraud Detection Dataset](https://www.kaggle.com/datasets/kartik2112/fraud-detection) from Kaggle. It contains historical credit card transactions, labeled as fraudulent (`is_fraud = 1`) or legitimate (`is_fraud = 0`).

## Project Structure
```text

├── data/                   # Place the downloaded datasets (fraudTrain.csv, fraudTest.csv) here
```

## Setup & Installation

1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

2. Download the data:
   Download the dataset from [Kaggle](https://www.kaggle.com/datasets/kartik2112/fraud-detection) and extract `fraudTrain.csv` and `fraudTest.csv` into the `data/` directory.

3. Run the Jupyter Notebook:
   ```bash
   jupyter notebook notebooks/fraud_detection.ipynb
   ```

## Workflow

1. **Data Preprocessing**: Handling missing values, dropping irrelevant features, and parsing temporal data.
2. **Exploratory Data Analysis (EDA)**: 
   - Analyzes fraud distribution.
   - Compares transaction amounts for fraud vs non-fraud.
   - Evaluates fraud occurrence across different categories and demographics.
3. **Feature Engineering & Encoding**: Categorical variable encoding (LabelEncoder) and feature scaling (StandardScaler).
4. **Model Training**: Logistic Regression, Decision Tree, Random Forest.
5. **Model Evaluation**: Metrics include Accuracy, Precision, Recall, F1 Score, Confusion Matrix, and ROC-AUC Score.
6. **Model Saving**: The best performing model is automatically saved to `model/fraud_model.pkl`.
